The features provided in the 'app/customer_registration/customer.py' code include:

1. Customer Creation API:
   - Validation of email, mobile number formats using custom validators.
   - Validation of business image URL format.
   - Uniqueness check to prevent active customers with duplicate email or mobile.
   - User authentication required (via OAuth2 token).
   - Logging with masked sensitive data.
   - Creation timestamp management.
   - Proper exception handling and response messaging.

2. List Customers API:
   - User authentication required.
   - Dynamic filtering on multiple fields such as customer_id, full_name, email, mobile, business_type, state, city, and is_active.
   - Supports inclusion/exclusion of inactive customers.
   - Pagination support with limit and offset.
   - Filtering by creation date range.
   - Error handling and logging.

3. Get Customer By ID API:
   - User authentication required.
   - Fetches customer by ID, returns 404 if not found.
   - Logging and error handling.

4. Filter Customers By Created Date API:
   - User authentication required.
   - Requires at least one of start or end creation date.
   - Filters only active customers.
   - Supports pagination.
   - Logging and error handling.

5. Edit Customer By ID API:
   - User authentication required.
   - Validation for full_name (non-empty), email, mobile with uniqueness checks excluding current customer.
   - Validation of business image URL format.
   - Optional update of business-related fields.
   - Requires at least one field to update.
   - Updates 'updated_at' timestamp.
   - Returns updated customer info with success message.
   - Logging and error handling.

6. Edit Customer By Mobile API:
   - Similar validations and update logic as Edit Customer By ID API, but identification via mobile number.
   - Ensures presence of at least one update field.
   - Logging and error handling.

7. Utility functions and configurations:
   - Masking sensitive data for logs.
   - Token validation using OAuth2.
   - Logging setup with formatting.
   - Usage of async PostgreSQL connection pool for DB operations.

Note: Rate limiting or request throttling is mentioned as a recommended improvement but not implemented in the current code.

Overall, the code provides RESTful CRUD operations for customers with authentication, validation, filtering, pagination, and logging.
